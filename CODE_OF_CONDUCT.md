# Code of Conduct

## Our Commitment

The UCLI Tools community is dedicated to providing a productive, inclusive, and professional environment for all contributors. We value technical excellence, constructive collaboration, cross-platform compatibility, and respectful communication in our shared mission to build high-quality command-line tools.

## Standards

### Expected Behavior

- **Be Professional** - Communicate clearly, respectfully, and constructively
- **Be Inclusive** - Welcome contributors from all backgrounds and skill levels
- **Be Helpful** - Share knowledge, provide guidance, and assist others when possible
- **Be Collaborative** - Work together toward improving CLI tools for all platforms
- **Be Respectful** - Acknowledge diverse perspectives and experiences
- **Give Credit** - Recognize others' contributions, ideas, and expertise
- **Test Thoroughly** - Ensure changes work across Linux, macOS, and Windows

### Unacceptable Behavior

- Personal attacks, insults, or derogatory comments
- Harassment of any kind (including but not limited to sexual harassment)
- Discrimination based on race, gender, sexual orientation, disability, or any other protected characteristic
- Trolling, deliberately disruptive behavior, or off-topic content
- Publishing others' private information without explicit permission
- Spam, excessive self-promotion, or inappropriate content
- Any conduct that creates an intimidating, hostile, or unwelcoming environment
- Technical bullying or dismissing others' valid concerns about compatibility

## Scope

This Code of Conduct applies to all UCLI Tools project spaces, including:

- GitHub repositories, issues, pull requests, and discussions
- Documentation, wikis, and README files
- Community repositories and contribution guidelines
- Tool READMEs and help documentation
- Any official UCLI Tools communication channels
- Community events, meetups, or online discussions about UCLI Tools

## Enforcement

Project maintainers are responsible for enforcing this Code of Conduct. They have the authority to:

- Remove, edit, or reject contributions that violate this code
- Temporarily or permanently ban contributors for repeated or severe violations
- Take any other action deemed appropriate to maintain a positive community environment
- Make final decisions on enforcement actions

## Reporting

If you experience or witness unacceptable behavior, please report it:

### How to Report

1. **Contact maintainers directly** - Tag `@ucli-tools/maintainers` in the relevant issue, discussion, or repository
2. **Open a private issue** - In the [community repository](https://github.com/ucli-tools/community/issues) with "Code of Conduct Report" in the title
3. **For serious violations** - Contact maintainers privately via their GitHub profiles or email

### What Happens Next

- All reports will be reviewed promptly and handled with discretion
- We will take appropriate action based on the severity and circumstances
- Reporters will be kept informed of the outcome (without compromising privacy)
- We prioritize the safety and well-being of our community members

### Anonymous Reporting

If you prefer to remain anonymous, you can:
- Create a GitHub issue from a throwaway account
- Contact maintainers through indirect channels
- Report through community discussions (though this may be less private)

## Consequences

Violations of this Code of Conduct may result in:

- **Warning** - Private or public warning about the behavior
- **Temporary Ban** - Temporary suspension from community spaces
- **Permanent Ban** - Permanent removal from all UCLI Tools spaces
- **Content Removal** - Removal of violating content
- **Other Actions** - As deemed appropriate by maintainers

Repeat violations or severe violations may result in more serious consequences.

## Appeals

If you believe an enforcement decision was made in error:

1. Contact the maintainers privately
2. Provide additional context or clarification
3. Appeals will be reviewed by the full maintainer team
4. Decisions are final but made with careful consideration

## Tool-Specific Considerations

Since UCLI Tools focuses on command-line utilities:

- **Cross-platform respect** - Don't dismiss compatibility concerns for any supported platform
- **CLI etiquette** - Be patient when helping others with command-line issues
- **Tool criticism** - Focus on technical merits, not personal attacks on tool authors
- **Testing expectations** - Acknowledge that thorough cross-platform testing takes time and effort

## Attribution

This Code of Conduct emphasizes merit-based technical contributions, professional conduct, and inclusive collaboration in the CLI tools ecosystem.

## Contact Information

For questions about this Code of Conduct or to report violations:

- **Primary Contact**: Open an issue in [ucli-tools/community](https://github.com/ucli-tools/community/issues)
- **Maintainers**: Check repository maintainer lists for direct contact options
- **Emergency**: For immediate safety concerns, contact GitHub support

---

**TL;DR:** Be kind, be professional, contribute based on merit, respect all platforms and users. We're here to build excellent CLI tools together in a welcoming community.
